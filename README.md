Hospital Information System (HIS) — Admin Dashboard

HIS Medical — це вебзастосунок для автоматизації адміністративної діяльності медичного закладу. Проєкт розроблений як інформаційна система для керування базою пацієнтів, штатом лікарів та графіком прийомів.

📋 Опис функціоналу
Система реалізована як SPA (Single Page Application) і дозволяє:
Керувати пацієнтами: Реєстрація нових відвідувачів та повне видалення карток з бази.
База спеціалістів: Перегляд списку лікарів, їхніх спеціалізацій, номерів кабінетів та графіків роботи.
Медичні послуги: Перегляд переліку послуг, закріплених за конкретними лікарями.
Життєвий цикл записів:
Створення запису (з вибором пацієнта, лікаря та фільтрацією послуг).
Зміна статусу: Заплановано → Прийнято.
Завершення візиту: переміщення запису в Архів.
Статистика: Дашборд із кількістю пацієнтів, лікарів та активних прийомів.
Пошук: Глобальний фільтр по всіх таблицях та картках у реальному часі.

🛠 Технологічний стек
Backend: Node.js, Express.js (REST API).
Database: MongoDB (використано ODM Mongoose).
Frontend: HTML5, CSS3 (Custom Theme, Inter Font), Bootstrap Icons.
Контейнеризація: Docker, Docker Compose.

🚀 Як запустити проєкт
Проєкт повністю контейнеризований. Вам не потрібно встановлювати Node.js, MongoDB чи інші інструменти локально (окрім Docker).
1. Запуск застосунку
Відкрийте термінал у кореневій папці проєкту та виконайте команду:

docker-compose up --build

Ця команда автоматично завантажить необхідні образи, створить базу даних та запустить сервер.

2. Виконання міграції бази даних
Після того, як контейнери запустяться (ви побачите повідомлення про успішне підключення до бази), необхідно заповнити систему початковими даними (лікарями та послугами).
Відкрийте нове вікно терміналу і виконайте:

docker-compose exec app node seed.js

🌐 Використані порти
Система використовує наступні порти:
Web-інтерфейс та API: http://localhost:5005
MongoDB (внутрішній): 27017 (доступний всередині Docker-мережі).

📖 Інструкція користувача
1.Реєстрація: Перейдіть у вкладку "Пацієнти" та створіть тестового користувача.
2.Створення запису: Перейдіть у "Записи" -> "+ Створити запис". Оберіть створеного пацієнта. Оберіть лікаря (дані підтягнуться з міграції). Оберіть послугу (система покаже лише ті послуги, які надає цей лікар).
3.Обробка прийому: У таблиці записів натисніть "Прийняти". Після завершення прийому натисніть "В архів".
4.Перегляд архіву: Завершений запис автоматично зникне з головної таблиці та з'явиться у вкладці "Архів".

📂 Структура проєкту
./models — схеми даних Mongoose (Patient, Doctor, Appointment, Service).
./routes — API ендпоінти для кожної сутності.
./public — фронтенд частина (HTML, CSS, клієнтський JS).
.server.js — головний файл сервера.
.seed.js — скрипт міграції (заповнення бази).
.Dockerfile & docker-compose.yml — конфігурація контейнеризації.
