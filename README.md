# Hospital Information System (HIS) — Admin Dashboard

**HIS Medical** — це вебзастосунок для автоматизації адміністративної діяльності медичного закладу. Проєкт розроблений як інформаційна система для керування базою пацієнтів, штатом лікарів та графіком прийомів.

---

## 📋 Опис функціоналу
Система реалізована як SPA (Single Page Application) і дозволяє:

*   **Керувати пацієнтами:** Реєстрація нових відвідувачів та повне видалення карток з бази.
*   **База спеціалістів:** Перегляд списку лікарів, їхніх спеціалізацій, номерів кабінетів та графіків роботи.
*   **Медичні послуги:** Перегляд переліку послуг, закріплених за конкретними лікарями.
*   **Життєвий цикл записів:**
    1.  Створення запису (з вибором пацієнта, лікаря та фільтрацією послуг).
    2.  Зміна статусу: **Заплановано** → **Прийнято**.
    3.  Завершення візиту: переміщення запису в **Архів**.
*   **Статистика:** Дашборд із кількістю пацієнтів, лікарів та активних прийомів.
*   **Пошук:** Глобальний фільтр по всіх таблицях та картках у реальному часі.

---

## 🛠 Технологічний стек
*   **Backend:** Node.js, Express.js (REST API).
*   **Database:** MongoDB (використано ODM Mongoose).
*   **Frontend:** HTML5, CSS3 (Custom Theme, Inter Font), Bootstrap Icons.
*   **Контейнеризація:** Docker, Docker Compose.

---

## 🚀 Як запустити проєкт
Проєкт повністю контейнеризований. Вам не потрібно встановлювати Node.js чи MongoDB локально.

### 1. Запуск застосунку
Відкрийте термінал у кореневій папці проєкту та виконайте команду:
```bash
docker-compose up --build
Ця команда завантажить образи, створить локальну базу даних та запустить сервер.
2. Виконання міграції бази даних
Після запуску контейнерів необхідно заповнити базу даних лікарями та послугами. Відкрийте нове вікно терміналу і виконайте:
code
Bash
docker-compose exec app node seed.js
🌐 Використані порти
Система використовує наступні порти:
Web-інтерфейс: http://localhost:5005
MongoDB: 27017 (внутрішній порт контейнера).
📖 Інструкція користувача
Реєстрація: Перейдіть у вкладку "Пацієнти" та зареєструйте нового користувача.
Створення запису: У розділі "Записи" натисніть "+ Створити запис". Оберіть пацієнта, лікаря та послугу.
Обробка прийому: Натисніть "Прийняти", щоб підтвердити візит. Після завершення натисніть "В архів".
Архів: Завершені записи автоматично переміщуються у вкладку "Архів".
📂 Структура проєкту
./models — моделі даних Mongoose.
./routes — API маршрути.
./public — фронтенд (HTML, CSS, JS).
server.js — точка входу в застосунок.
seed.js — скрипт початкового заповнення бази.
Dockerfile & docker-compose.yml — конфігурація Docker.
